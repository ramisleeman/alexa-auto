Detailed Description

MediaPlayer should be extended to play audio data provided by the Engine. MediaPlayer is the interface for audio playback for an AudioChannel. After returning true from a playback-controlling directive from the Engine (i.e. play(), pause(), stop(), resume()), the platform implementation should notify the Engine when the platform media player changes playback state by calling mediaStateChanged() with the new MediaState. The Engine expects no call to mediaStateChanged() for a directive in which the platform returned false. The platform implementation may call mediaError() or mediaStateChanged() with MediaState.BUFFERING at any time during a playback operation to notify the Engine of an error or buffer underrun, respectvely. When the media player resumes playback after a buffer underrun, the platform implementation should call mediaStateChanged() with MediaState.PLAYING.

Note
The MediaPlayer platform implementation should be able to support the audio formats recommended by AVS for a familiar Alexa experience: https://developer.amazon.com/docs/alexa-voice-service/recommended-media-support.html

◆ isRepeating()

boolean com.amazon.aace.alexa.MediaPlayer.isRepeating ( )

Checks if playback of the current audio source must be repeated when finished playing. If this call returns true, the platform implementation should repeat the current audio source. If false is returned, the platform implementation should call mediaStateChanged() with MediaState.STOPPED when the audio is finished playing. Note that when stop has been called, audio source should stop repeating regardless of this value.
